================================================================================
DIRECTORY DESCRIPTIONS
================================================================================

data/
  Purpose: Store all source documents, processed data, and vector database
  Subdirectories:
    - raw/           : Original PDF and text files (organized by department)
    - processed/     : Chunked text and metadata after preprocessing
    - vector_db/     : ChromaDB persistent storage (SQLite backend)

src/
  Purpose: Core source code for RAG system components
  Subdirectories:
    - config/        : Configuration files (model names, paths, parameters)
    - ingestion/     : Document loading and preprocessing
    - embeddings/    : Embedding generation using sentence-transformers
    - vectorstore/   : ChromaDB operations (indexing, retrieval)
    - retrieval/     : Query processing and similarity search
    - generation/    : LLM integration (Ollama) and response generation
    - utils/         : Logging, validation, and helper functions

pipeline/
  Purpose: High-level orchestration of RAG workflows
  Files:
    - ingest_pipeline.py  : Document ingestion → chunking → embedding → indexing
    - query_pipeline.py   : Query → retrieval → context assembly → generation
    - orchestrator.py     : End-to-end RAG coordination

tests/
  Purpose: Unit and integration tests for all components
  Coverage:
    - PDF extraction, text parsing, chunking
    - Embedding generation, vector storage
    - Retrieval accuracy, LLM generation
    - End-to-end pipeline validation

evaluation/
  Purpose: System evaluation and performance metrics
  Files:
    - metrics.py         : Context relevance, answer accuracy, faithfulness
    - test_queries.json  : Sample queries across Finance, HR, Manufacturing
    - evaluate.py        : Run evaluation suite

scripts/
  Purpose: Automation scripts for setup and execution
  Files:
    - setup_environment.sh : Install WSL2, Ollama, dependencies
    - download_models.sh   : Download Mistral 7B and embedding model
    - run_ingestion.py     : Execute document ingestion pipeline
    - run_query.py         : CLI interface for querying the system

notebooks/
  Purpose: Jupyter notebooks for experimentation and analysis
  Files:
    - Data exploration (inspect PDFs and text datasets)
    - Chunking experiments (test different chunk sizes/overlaps)
    - Embedding analysis (visualize vector space)
    - Retrieval testing (evaluate retrieval quality)

docs/
  Purpose: Project documentation
  Files:
    - architecture.md        : System design and component interactions
    - setup_guide.md         : Installation instructions (Windows + WSL2)
    - usage_guide.md         : How to run ingestion and query pipelines
    - evaluation_results.md  : Performance metrics and test results

================================================================================
EXECUTION FLOW
================================================================================

1. SETUP PHASE
   ├── Install WSL2 (scripts/setup_environment.sh)
   ├── Install Python dependencies (pip install -r requirements.txt)
   ├── Download models (scripts/download_models.sh)
   └── Start Ollama server (ollama serve in WSL2)

2. INGESTION PHASE (scripts/run_ingestion.py)
   ├── src/ingestion/pdf_extractor.py     → Extract text from PDFs
   ├── src/ingestion/text_parser.py       → Parse text datasets
   ├── src/ingestion/serializer.py        → Convert records to text
   ├── src/ingestion/chunker.py           → Split into chunks
   ├── src/embeddings/generator.py        → Generate embeddings
   └── src/vectorstore/indexer.py         → Store in ChromaDB

3. QUERY PHASE (scripts/run_query.py or pipeline/query_pipeline.py)
   ├── src/retrieval/query_processor.py   → Process user query
   ├── src/retrieval/retriever.py         → Retrieve relevant chunks
   ├── src/generation/prompt_builder.py   → Build LLM prompt
   ├── src/generation/llm_client.py       → Call Ollama API
   └── src/generation/response_handler.py → Format and return response

4. EVALUATION PHASE (evaluation/evaluate.py)
   ├── Load test queries (evaluation/test_queries.json)
   ├── Run queries through pipeline
   ├── Calculate metrics (evaluation/metrics.py)
   └── Generate report (evaluation/evaluation_results.md)


================================================================================
NOTES
================================================================================

- All paths are relative to project root (unido-enterprise-rag/)
- data/raw/ contains original files (never modified)
- data/processed/ contains intermediate outputs (can be regenerated)
- data/vector_db/ contains ChromaDB persistent storage (disk-backed)
- src/ contains reusable modules (importable across scripts)
- pipeline/ contains high-level orchestration (calls src/ modules)
- tests/ mirrors src/ structure for easy test discovery
- scripts/ contains executable entry points (.py and .sh)
- notebooks/ for experimentation only (not production code)
- Configuration centralized in src/config/settings.py and .env

Total Structure:
  - 7 main directories
  - 13 subdirectories
  - ~40 Python files
  - ~10 support files (docs, config, scripts)

Hardware Optimized For:
  - Windows 64-bit + WSL2
  - Intel i5-1235U (CPU-only processing)
  - 8GB RAM (memory-efficient components)
  - 289GB storage (ChromaDB persistent mode)
